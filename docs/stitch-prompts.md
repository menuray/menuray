# Happy Menu — Stitch-Ready Prompt 包

> 用法：
> 1. 在 Stitch 新会话中先丢 **「系统提示」**，定下整体调性。
> 2. 接着按需丢 **「单屏 prompt」**，每屏一份独立提示。
> 3. 需要微调时用末尾 **「迭代 prompt 模板」**。

---

## 一、系统提示（每次新会话开头丢这一份）

```
我在设计一个叫 Happy Menu 的应用。

Slogan：拍一张照，5 分钟生成电子菜单。

定位：商家拍一张纸质菜单照片，5 分钟生成一份可分享的电子菜单（链接 / 二维码 / PDF）。不做点餐和支付，纯展示与分发。

目标用户：
- 商家端：中小型餐厅老板 / 店长，35–55 岁，没空也不懂排版。
- 顾客端：到店扫码 或 朋友分享链接 看菜单的人，包含中外语言用户。

核心功能：拍照 / 上传 → OCR 识别 → 结构化菜品 → 编辑修正 → AI 配图 / 翻译 / 描述扩写 → 模板化电子菜单 → 链接 / 二维码 / PDF 分发 → 多套菜单切换（午 / 晚 / 季节）→ 售罄一键切换 → 数据统计。

平台：商家端为移动端原生（iOS / Android），顾客端为响应式 H5。

== 品牌系统（所有屏幕严格遵守）==

品牌色：
- Primary    #2F5D50  墨绿     用于品牌标识、主导航、主按钮、强调
- Accent     #E0A969  琥珀金   用于 CTA 高亮、重要数据、点缀
- Surface    #FBF7F0  暖米白   主背景
- Ink        #1F1F1F  深炭     主文字
- Secondary  #6B7B6F  灰绿     次要文字
- Success    #4A8A6E  墨绿衍生
- Warning    #E0A969  与 Accent 同色或加深
- Error      #C2553F  砖红
- Divider    #ECE7DC  暖灰

字体：
- 中文：思源黑体 / OPPO Sans 同款气质（现代无衬线，规整易读）
- 英文：Inter / DM Sans 同款气质
- 标题略重 (600–700)，正文 regular，价格用半重号

圆角与间距：
- 卡片 16px，按钮 12px，输入框 10px，小标签 chip 999px (pill)
- 8px 基线网格

风格基调：
- 商家端：现代简洁、清晰高效、专业可信，重操作易用 — 大触控目标、清晰层级
- 顾客端：餐饮温暖感、食物诱人、信息清晰 — 大图、温暖光感
- 整体气质：稳重 + 温暖 + 一点琥珀金的高级感

Logo：
- 方形 App Icon
- 背景：墨绿 #2F5D50
- 主体：暖米白色的菜单矩形，居中略偏左
- 菜单的右下角向上翻卷，卷起的内角呈现琥珀色 #E0A969 高光
- 整体微圆角，无文字
- 在小尺寸下（32px）仍能识别"翻起一角"的剪影

品牌出现位置：
- 商家端 App 启动页 / 登录页 / 我的页 顶部出现 Logo + "Happy Menu"（两个单词中间留空格）+ slogan
- 顾客端菜单页底部小字："由 Happy Menu 提供"
- 品牌名英文为 "Happy Menu"（两单词分写），不要把 "Happy" 翻译成中文
- 排版可做：水平 "Happy Menu"、上下堆叠 "Happy / Menu"、或 "Happy" 用琥珀色 + "Menu" 用墨绿
- URL / handle 用连写：happymenu.com / @happymenu（仅用于域名场景）

所有屏幕严格保持视觉系统一致：同一套配色、字体、圆角、间距规范。
```

## 二、商家端 17 屏

### A1 登录 / 注册

```
屏幕：商家登录 / 注册
目的：让餐厅老板用手机号 + 验证码快速进入。

用户操作：
- 输入手机号
- 收 / 输验证码
- 切换"登录 / 注册"

页面元素：
- 顶部品牌 Logo + 一句 slogan：「拍一张照，5 分钟生成电子菜单」
- 手机号输入框、验证码输入框 + "获取验证码" 按钮（含 60s 倒计时）
- 主按钮："登录"
- 切换链接："新用户？立即注册"
- 底部：用户协议 + 隐私政策链接

状态：
- 验证码倒计时进行中（按钮灰色 + "59s 重发"）
- 验证码错误（红色提示）
- 登录成功 → 跳转 A2 首页

风格基调：温暖、简洁、信任感。
```

### A2 首页 / 菜单列表

```
屏幕：商家首页 - 我的菜单
目的：展示所有已创建菜单，支持快速进入 / 新建。

用户操作：
- 点 "+ 新建菜单" 进入 A3
- 点某张菜单卡片进入 A14 管理
- 搜索菜单

页面元素：
- 顶部店铺名 + 头像 / 切换门店入口
- 搜索框
- "+ 新建菜单" 浮动按钮 (FAB)
- 菜单卡片网格：缩略图 + 菜单名 + 状态标签（"已发布" / "草稿"）+ 最近更新时间 + 访问量小数字
- 卡片右上更多菜单（重命名 / 复制 / 删除）

示例数据：
- "午市套餐 2025 春" - 已发布 - 3 天前 - 1,247 次访问
- "晚市菜单" - 已发布 - 1 周前 - 893 次访问
- "周末早午餐" - 草稿 - 今天

状态：
- 空状态：插画 + "还没有菜单，拍一张试试" + 大按钮 "立即新建"
- 加载中：骨架屏

底部 Tab：菜单 / 数据 / 我的
```

### A3 新建菜单 - 拍照

```
屏幕：拍摄菜单照片
目的：调用相机拍下纸质菜单，可多张连拍。

用户操作：
- 按快门拍照
- 切换闪光灯
- 切换到"上传相册"模式（A4）
- 完成进入 A5 预处理

页面元素：
- 全屏取景框
- 实时菜单边缘检测（黄色虚线框跟随）
- 顶部：关闭按钮、闪光灯切换、"换相册上传"
- 底部：已拍数量小缩略图条 + 大快门按钮 + "完成 (N)" 按钮
- 中部提示气泡："对准菜单边缘，自动校正"

状态：
- 边缘检测成功：黄框 → 绿框 + 提示 "可以拍了"
- 光线不足：顶部黄色提示条 "光线偏暗，建议开启闪光灯"
- 已拍 5 张：底部显示 5 张缩略图，可左右滑动查看 / 删除

示例：缩略图区显示 3 张拍好的菜单页。
```

### A4 新建菜单 - 上传

```
屏幕：从相册上传菜单
目的：让商家从本地选 1 张或多张菜单图。

用户操作：
- 多选图片
- 调整选中顺序
- 进入 A5 预处理

页面元素：
- 顶部：取消、标题 "选择菜单图片"、右上 "下一步 (N)"
- 相册图片网格（每行 4 张）
- 已选中图片右上角显示选择序号 ①②③...
- 底部已选缩略图条，可拖拽排序

示例：
- 第 1 张选中（标 ①），第 4 张选中（标 ②），第 7 张选中（标 ③）
- 底部缩略图条：3 张菜单照片，可拖动调整顺序

状态：
- 未选中：右上 "下一步" 灰色
- 选中至少 1 张：按钮高亮，显示数量
```

### A5 图像预处理

```
屏幕：图像校正与裁剪
目的：让商家逐张校正菜单照片的透视、裁剪、旋转。

用户操作：
- 拖动 4 角点调校透视
- 旋转 90°
- 重置 / 自动校正
- 切换到下一张
- 全部完成 → 进入 A6 OCR

页面元素：
- 顶部：返回、标题 "校正图片 (1 / 3)"、右上 "下一步"
- 中部大图 + 4 个可拖动角点 + 校正后预览叠层
- 底部工具栏：自动校正、旋转、裁剪、对比度增强、撤销
- 底部缩略图条：当前编辑张高亮

状态：
- 自动校正 loading：菱形动画 + "智能校正中"
- 校正完成：缩略图角标变绿勾
- 翻页过渡：左右滑动切换

示例：菜单四角已自动检测，显示绿色辅助点。
```

### A6 OCR 识别中

```
屏幕：识别处理中
目的：在 OCR 解析时给商家清晰的进度反馈，避免焦虑。

用户操作：仅查看（可后台运行 / 取消）

页面元素：
- 居中插画：菜单图 → 文字流动 → 菜品卡片，循环动画
- 进度文案分阶段：
  1. "正在识别文字... (1 / 3 张)"
  2. "正在识别菜品结构..."
  3. "正在归类整理..."
- 进度条
- 底部小字提示："识别用时约 30 秒，可点'后台运行'继续操作"
- 主按钮："后台运行"  次要："取消"

状态：
- 识别完成：进度条满 → 自动跳 A7
- 识别失败：进度条变红 + "识别失败，请重试" + "重试 / 返回" 按钮

风格：温暖动态，缓解等待焦虑。
```

### A7 识别结果编辑

```
屏幕：识别结果 - 菜品列表编辑
目的：让商家快速核对、修正 OCR 识别出的菜品。

用户操作：
- 行内编辑菜名 / 价格
- 长按拖拽排序
- 滑动删除
- 点菜品行进入 A8 详情编辑
- 顶部 "+ 新增类别 / 菜品"
- 完成 → 进入 A10 模板选择

页面元素：
- 顶部：返回、标题 "整理菜单"、右上 "下一步"
- 类别折叠列表（可点收起 / 展开）
- 每行菜品：菜名 + 价格 + 小图标（标签提示 / 翻译已加 / 配图已加）
- 低置信度行：左侧橙色竖条 + 末尾 "?" 提醒
- 浮动按钮："+ 新增"

示例数据：
- 类别 "凉菜"
  - 口水鸡 ¥38
  - 凉拌黄瓜 ¥18
  - **? 川北凉粉 ¥22**（低置信度橙色条）
- 类别 "热菜"
  - 宫保鸡丁 ¥48
  - 麻婆豆腐 ¥32

状态：
- 编辑某行：行变蓝边、底部弹键盘
- 拖拽排序：行半透明跟随手指
- 空类别：显示 "+ 添加菜品"
```

```

- 价格输入（含货币符）
- 描述多行输入 + 右下 "AI 扩写"

- 名称：宫保鸡丁
- 价格：¥48

- 标签：招牌、推荐
- 过敏原：花生

状态：
- AI 扩写中：按钮转圈 + "AI 扩写中..."
- 一键翻译完成：EN 输入框淡入
```

### A9 AI 增强工具

```

目的：让商家一次性给整份菜单加图、加描述、加翻译。

- 主按钮："开始增强"
- 进度态：分项进度条

示例：

状态：
- 进行中：每项卡片显示子进度
- 完成：显示绿色勾 + "已增强 23 道菜，查看结果"
- 部分失败：橙色 + "3 道失败，重试"
```

### A14 菜单管理（已发布菜单的统一入口）

```

目的：进入某个已发布菜单后的统一管理入口。

- 顶部：返回、菜单名（可编辑）、右上更多菜单
```

## 五、使用建议

1. **先丢系统提示，再生成首页**（A2 或 B1），看 Stitch 给出的整体调性是否合你意；不满意就先在系统提示里调风格关键词，别急着生成所有屏。
2. **同一会话尽量做完一类**（比如"先把商家端 17 屏全部生成"），让 Stitch 在同一上下文中保持视觉一致。
3. **示例数据保留**：里面的 "云间小厨"、"宫保鸡丁 ¥48" 这些都是有意写的，不要换 Lorem Ipsum，否则界面看起来会很假。
4. **状态变量分别生成**：空状态 / 加载中 / 错误态，建议分别用迭代 prompt 单独生成，得到的截图集会更完整。
5. **导出**：满意后用 Stitch 自带的 Figma 导出或代码导出。
